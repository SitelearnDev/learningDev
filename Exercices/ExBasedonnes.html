Exercices Base de donnée

Exercice 1 : Création d'une table

Créez une table "Utilisateurs" avec les colonnes suivantes : ID (clé primaire) Nom Adresse e-mail

Exercice 2 : Insertion de données

Insérez trois utilisateurs fictifs dans la table que vous venez de créer.

Exercice 3 : Requête simple

Écrivez une requête SQL pour sélectionner tous les utilisateurs de la table.

Exercice 4 :Filtrage de données

Modifiez votre requête pour sélectionner uniquement les utilisateurs dont le nom est "Smith".

Exercice 5 : Jointure simple

Créez une nouvelle table "Commandes" avec les colonnes suivantes : ID (clé primaire) ID_utilisateur (clé étrangère faisant référence à la table Utilisateurs) Montant

Exercice 6 : Requête avec jointure

Écrivez une requête SQL pour sélectionner le nom de l'utilisateur et le montant de toutes les commandes.

Exercice 7 : Agrégation de données

Modifiez votre requête pour calculer le montant total des commandes pour chaque utilisateur.

Exercice 8 : Sous-requête:

Écrivez une requête SQL pour sélectionner tous les utilisateurs qui ont passé une commande.

Exercice 9 : Contraintes et index

Ajoutez une contrainte d'unicité sur l'adresse e-mail dans la table Utilisateurs, puis créez un index sur la colonne Nom.

Exercice 10 : Requête complexe

Écrivez une requête SQL pour sélectionner les utilisateurs qui ont passé des commandes d'un montant supérieur à la moyenne de tous les montants de commande.

Exercice 11 :

1. Créer la base de donnée en précisant les clés primaires et étrangères.
2. Remplir les tables par des données.
3. Créer un compte de connexion ‘visiteur’ avec le mot de passe ‘1234’ et la base de données par défaut ‘ABONNEMENTS’.
4. Créer une vue AbonClt qui affiche pour chaque client (nom) les types et les numéros de téléphone auxquels il est souscrit.
5. Interdire à ‘visiteur’ l’accès en sélection et modification à toutes les tables sauf à la vue AbonClt.
6. Connecter vous en tant que ‘visiteur’ et tester.
7. Reconnecter vous avec votre session initiale et créer un rôle ‘Agent’, attribuer lui le droit de sélection sur la table Client, et retirer lui ce droit de la table Abonnement puis interdire la sélection à ‘Agent’ pour la table Souscrire.
8. Créer un utilisateur ‘Agent1’ avec le mot de passe ‘4321’ et la base de données par défaut ‘ABONNEMENTS’.
9. Ajouter l’utilisateur ‘Agent1’ au rôle ‘Agent’.
10. Attribuer à l’utilisateur ‘Agent1’ le droit de sélection sur la table Abonnement et Souscrire.
11. Connectez vous en tant que ‘Agent1’ et essayez d’afficher le type et tarif des abonnements puis le Nclt, le Code et le numéro de téléphone des clients, qu’est ce que vous remarquez et comment vous expliquez les résultats ?

Exercice 12 :

Villes de France:
* Créez une base de données pour stocker des informations sur les villes de France.
* Ajoutez une table pour les villes avec des colonnes telles que le nom de la ville, la population, la région, etc.
* Remplissez la table avec quelques villes fictives.
* Écrivez des requêtes SQL pour afficher les noms des villes et leur population.
Système de commandes:
* Imaginez une base de données pour un système de commandes en ligne.
* Créez des tables pour les clients, les produits, les commandes, etc.
* Ajoutez des enregistrements factices dans chaque table.
* Écrivez des requêtes SQL pour afficher les détails des commandes, les produits commandés par un client donné, etc.
QCM (Questionnaire à choix multiples):
* Créez une table pour stocker des questions et leurs réponses possibles.
* Ajoutez quelques questions et réponses factices.
* Écrivez des requêtes SQL pour afficher les questions et leurs réponses.
